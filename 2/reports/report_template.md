# ДЗ2 — Отчёт (шаблон)

## 1. Описание среды
- Название: `logic_circuit`
- Почему задача подходит под требования:
  - Верифицируемая: ответ проверяется детерминированной симуляцией схемы.
  - Регулируемая сложность: число входов/вентилей и набор операций зависят от `difficulty`.
  - One-shot: все данные для решения в одном промпте, без интеракции со средой.

## 2. Маппинг сложности
Опишите соответствие `difficulty ∈ [1,10]` и гиперпараметров (`num_inputs`, `num_gates`, `allowed_gates`).

## 3. Датасеты
- Train: семплируемый (iterable), распределение сложностей.
- Test: фиксированные `test_d{d}.jsonl` для нескольких `d`, с фиксированными seed.

## 4. Обучение
- Модель: Qwen2.5-1.5B-Instruct
- Алгоритм: GRPO (reward = correctness)
- Ускорение: unsloth (FastLanguageModel)

## 5. Оценка
- Инференс: vLLM
- Метрика: accuracy (доля задач, где `Env.verify=True`)
- График: парные бары baseline vs finetuned по `difficulty`

## 6. Результаты и анализ (самое важное)
Ответьте:
- где улучшение максимальное/минимальное и почему?
- где модель “ломается” (ошибки парсинга, логика, длинный контекст)?
- как влияет сложность (num_gates/ops) на качество?
- какие абляции/эксперименты вы бы сделали дальше?
